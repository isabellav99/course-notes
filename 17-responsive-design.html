Responsive Design

Being Flexible

Up to this point, you may have created web sites which are pixel-perfect. WYSIWYG.

The web is a fluid medium. We never know how a person may view our site, so we need to modify our designs to be flexible.

We should aim to design sites which look good on big and small screens alike.

Think about media

Benefits of Responsive Design

Responsive, fluid designs use one HTML file and adjust the styles based on the window/device size.

  • ESPN

@media queries

@media queries allow a designer to specify styles based on conditions.

  • device-height
  • device-width
  • height
  • orientation
  • resolution
  • width

Styling for Mobile

Two components are necessary to adjust a design for mobile:

  • a meta tag specifying viewport information
  • a style sheet which includes a @media query for a given breakpoint

Styling for Mobile: HTML (step 1)

<meta name="viewport" content="width=device-width; initial-scale=1;" />

Without this tag, mobile designs will not work.

Maximum Scale

Some mobile web tutorials suggest adding the following to the meta tag: maximum-scale=1.

DO NOT include this attribute.

It disables the pinch-zoom feature on mobile devices, making the design less accessible.

Styling for Mobile: HTML (step 2)

Add a link to the mobile style sheet AFTER the style sheet for the web.

<link href="css/mobile.css" rel="stylesheet"
media="screen" />

Don't Be Fooled

There is a media type: handheld. Today's mobile operating systems (iOS, Android) do not support it. Do not use it.

Apple and Google market their devices as being able to view the web — as-is — and disabled the use of the handheld media type.

(This is why the screen media type is used and we set widths and/or orientation to create designs for mobile.)

Styling for Mobile: CSS

Mobile styles should always come after the styles set for a desktop version.

It is generally a good idea to keep desktop, mobile styles separate.

Start a new mobile style sheet with the following:

@media (max-width: 320px) {
  /* mobile styles go here! */
}

Note: '320' may be any value.

Modifying a Design for Mobile

Mobile design involves overwriting CSS rules created for the web version of a site.

If you do not overwrite rules from your main (web) css file, the mobile version of the site will inherit those rules.

Do not start building a mobile version until you have finalized a web version.

Transitioning a Design

Different designs pose different challenges. Always think critically about a site's content, design.

  • Increase font size
  • Remove floats, positioning (get everything in one column)
  • Hide elements not necessary for mobile
  • Add mobile-only content if necessary
